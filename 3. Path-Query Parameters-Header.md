**Resource:** Resource represent API/Collection which can be accessed from the Server

Example google.com ithu server ithula Maps, Docs, Search, Chat, Images, etc.. intha apps la API itha "Resource" nu solluvanga

Nama oru Resuest send pandrom na parameters potu call pandra situation varum pothu athula rendu irukku

  * Path Parameter: ippo google.com server la Map API/Resource call pandra ok.. athula ethum specefic ah venum - specefic ah mention panni call pandrathukku path parameter
  use pandrom. Inga Map API/Resource Collections aagiduthu. example advanced Xpath maari

     Example: google.com/maps/**12345**

  * Query Parameter: Ithu Sort/Filter the resource on the Server

     Example: https://amazon.com/orders?sort_by=12/29/2025

Ithu Query/Path parameter ah nu find panna

Parameter name and value ku munnadi "?" iruntha athu Query parameter illana Path parameter

END POINT/Base URL: Address where API is hosted on the server

 END POINT Structure: www.google.com(Base URL)/maps(Resource)/(Query/Path) Parameters

**Headers/Cookies:** It represents the meta-data associated with the API request and response, solla pona API ku nama send pandra request ah process pana additional 
details kudukurom ithu thaan, example Authorization details

**Header:** Nama API ku oru request send pandrom na antha request pathina detalils ithula tha kudupom. example ithu JSON format ah XML format ah innum related details la 
ithula kudupom

**Cookies:** API cookies, often referred to as HTTP cookies or web cookies, are small pieces of data that a web server sends to a client's browser and are stored on the client's device. These cookies are used to track and store information about the user's interactions with a website or web application. They play a crucial role in maintaining user sessions, personalizing experiences, and enhancing security.
