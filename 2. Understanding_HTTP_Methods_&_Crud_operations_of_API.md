SOAP - Heavy weight protocol

REST - Light weight protocol, have some leverages, it is easy to build and maintain, all the latest organizations like FB, Insta using REST

In REST there are some widely using methods like

  * GET: Oru information nama server la irunthu data fetch/read panna use pandrom - Just retrive/read only(GET is used to request data from a specified resource). **Idempotent**
    
  * POST: Server la pudhu data create panna/submit panna use pandrom. Uses = Form submit, User create, Order place, File upload, Login request(POST is used to send data to a server to create/update a resource).       **Non-Idempotent**
    
  * PUT: Existing resource ah full ah update or replace panna use pandrom(PUT is used to send data to a server to create/update a resource). **Idempotent**
    (PUT la usually full object anupanum PATCH maari use panna existing resource ah NULL or remove panidum. Full update, Replace resource, Idempotent, Full body)
    
  * PATCH: Existing resource la specefic fields mattum update panna use pandrom. Lightweight updates(PATCH method is used to apply partial modifications to a resource). **Idempotent depends**
    (Partial update, Modify fields, Usually idempotent, Partial body)
    
  * DELETE: Ithu oru existing information or resource ah delete pandrathukku ithu use panuvom(DELETE method deletes the specified resource). **Idempotent**
    
  * HEAD: Ithu GET maari tha but header mattum return pannum body data panathu(HEAD is almost identical to GET, but without the response body).
    
  * OPTIONS: Enna enna operations allowed nu capabilities check pandrathukku(The OPTIONS method describes the communication options for the target resource).
    
  * CONNECT: Proxy server kitta "Remote server kooda tunnel open pannu" nu solra request, Mainly HTTPS secure connection establish panna use aaguthu (The CONNECT method is used to start a two-way communications       tunnel) with the requested resource. (Normal = Client -> Server) and Praxy iruntha (Client -> Proxy -> Server).
    
  * TRACE: Client anupina request exact ah server kitta poi epdi reach aaguthu nu trace/echo back panna use panura method(The TRACE method is used to perform a message loop-back test that tests the path for the         target resource (useful for debugging purposes))

**Idempotent** - Same request ah 1 time send pannalum... 10 times send pannalum server result same ah iruntha Idempotent

Ithu major ah use pandrathunala itha nama CRUD nu solluvom (Create Retrive Update Delete)
